Vulnerability: Server-Side Request Forgery (SSRF)

  • The server takes your URL and makes an HTTP request to it server-side.
  • The internal admin API is only accessible from localhost: http://127.0.0.1/admin-api
  • Enter that URL as the target — the server will request it on your behalf, bypassing any external firewall.
  • Try also: http://localhost/admin-api

Webhook Tester

HARD +100 EXP

Send a webhook payload to an external server. The server will make an HTTP request to the URL you provide. Is the internal network protected? Try requesting something from localhost — the internal admin API at http://127.0.0.1/admin-api is not accessible from outside, but the server can reach it!
