Vulnerability: Reflected XSS (Cross-Site Scripting)

  • The server reflects the name parameter directly into the HTML page without sanitization.
  • Try entering: <script>alert(1)</script> in the input box.
  • Or try: <img src=x onerror="alert(document.cookie)">
  • The generated greeting page will open in a new tab — look at the source and check for the flag message in the page body.
  • The flag will be revealed in the greeting page once you successfully trigger the reflection.

Reflected Injection Point

MEDIUM +100 EXP

The server generates a greeting page reflecting exactly what you provided as the name parameter in the URL. Can you force it to execute an alert or JavaScript logic? The server does not sanitize the input at all.
