Vulnerability: SQL Injection (In-band)

The backend query looks like: SELECT * FROM users WHERE username = '[INPUT]'

  • Try typing: ' OR '1'='1 — This breaks the string and makes the condition always true.
  • Or try: ' OR 1=1-- — The -- comments out the rest of the query.
  • Look for the admin user row — their password field contains the flag!
  • The flag is the admin's password value in the database response.
