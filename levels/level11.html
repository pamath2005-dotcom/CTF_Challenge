Vulnerability: Cross-Site Request Forgery (CSRF)

  • The email change endpoint has no CSRF token — it accepts any POST from any origin.
  • A rogue website could silently submit this form in the background when a logged-in user visits it.
  • Click "SIMULATE_CSRF_ATTACK()" — this mimics a cross-site forged request being sent automatically.
  • The flag will appear in the server's response when the attack simulation succeeds.
  • Normal email changes do not return a flag — only the CSRF-simulated request does.
